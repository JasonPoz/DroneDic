# Терминологический словарь для БАС (Django-проект)

Веб-приложение на Django для работы с терминологией в области беспилотных авиационных систем.  
Основная фича — **поиск терминов** с учётом языка, опечаток и выводом **термина, определения и контекста** на паре языков:  
`язык источника → язык перевода`.

---

## 1. Технологии и стек

**Языки и фреймворки**

- Python 3.13
- Django 5.x
- HTML, CSS, JavaScript
- Bootstrap 5 (через шаблоны)

**Базы данных и хранилище**

- PostgreSQL (основная БД Django)

**Библиотеки и инструменты (основные)**

- `psycopg` — драйвер PostgreSQL
- `deep-translator` (`GoogleTranslator`) — машинный перевод термина/определения/контекста
- `difflib.get_close_matches` — поиск с учётом опечаток (на 1 символ)
- `django-bootstrap5` — интеграция Bootstrap в шаблонах
- Класс-based view `ListView` (Django) — для страницы поиска
- Django ORM — работа с моделями `TermTranslation`, `Definition`, `Context`, `Language`

> В проекте также используется модель BERT  
> **`paraphrase-multilingual-MiniLM-L12-v2`** (через `sentence-transformers`)  
> для эмбеддингов, но текущий интерфейс поиска терминов работает на прямом и fuzzy-поиске в БД.

### 2.6. Создание языков (таблица `Language`)

Если в базе ещё нет записей в `Language`, их нужно добавить.

Открой Django shell:

```bash
python manage.py shell
```

Внутри shell:

```python
from dictionary.models import Language

LANGUAGES = [
    {'code': 'ru', 'name': 'Русский'},
    {'code': 'en', 'name': 'Английский'},
    {'code': 'fr', 'name': 'Французский'},
    {'code': 'es', 'name': 'Испанский'},
    {'code': 'cn', 'name': 'Китайский'},
]

for item in LANGUAGES:
    Language.objects.get_or_create(code=item["code"], defaults={"name": item["name"]})
```

Также через админку или отдельные скрипты должны быть загружены:

* Термины (`Term`)
* Переводы терминов (`TermTranslation`)
* Определения (`Definition`)
* Контексты (`Context`)

Без этих данных поиск просто не найдёт ничего.

## 3. Как пользоваться поиском

### 3.1. Главная страница

На главной (`/`) есть:

* Поле ввода запроса (термина)
* Два выпадающих списка выбора языка:

  * **Слева** — язык, на котором вводится запрос (**язык источника**, `source_lang`)
  * **Справа** — язык, на который нужно получить результат (**язык перевода**, `target_lang`)
* Кнопка **Поиск**

Поддерживаются языки (если они заведены в таблице `Language`):

* Русский (`ru`)
* Английский (`en`)
* Французский (`fr`)
* Испанский (`es`)
* Китайский (`cn`)

### 3.2. Запуск поиска

1. Выбери **язык источника** (например, `Русский`).
2. Выбери **язык перевода** (например, `Английский`).
3. Введи термин (например, `дрон` или `система`).
4. Нажми кнопку **Поиск**.

Запрос отправится на URL вида:

```text
/search/?query=<слово>&source_lang=<код>&target_lang=<код>
```

Например:

```text
/search/?query=дрон&source_lang=ru&target_lang=en
```

### 3.3. Как работает поиск

Поиск реализован во вьюхе:

```python
class TermSearchView(ListView):
    ...
```

и использует модель `TermTranslation`:

* Сначала выбираются все переводы на выбранном **языке источника** (`language__code=source_lang`).
* Затем выполняется:

  1. **Прямой поиск** по названию термина:

     ```python
     name__icontains=query
     ```
  2. Если прямых совпадений нет — используется
     **поиск с учётом опечаток** через `difflib.get_close_matches`:

     ```python
     from difflib import get_close_matches

     close_matches = get_close_matches(query, names, n=50, cutoff=0.7)
     ```

     Это позволяет находить термины при ошибке в 1 символ (например, `систеа` вместо `система`).

### 3.4. Страница результатов поиска

Ответ отрисовывается шаблоном `pages/search_results.html`.

Для каждого найденного термина выводится **карточка**, содержащая:

* Заголовок:
  `Источник → Перевод`, например:
  **Русский → Английский**
* Блок **"Термин"**:

  * Термин на исходном языке
  * Термин на языке перевода (если есть в БД)
* Блок **"Определение"**:

  * Определение на исходном языке
  * Определение на языке перевода (если есть)
* Блок **"Контекст"**:

  * Контекст на исходном языке
  * Контекст на языке перевода (если есть)

Между карточками есть визуальные отступы и разделители, чтобы результаты не сливались.

### 3.5. Переключение языков перевода

Под каждой карточкой есть **кнопки других языков**, например:

* EN
* FR
* ES
* CN

Эти кнопки позволяют **переоткрыть поиск** для той же комбинации:

* Тот же запрос `query`
* Тот же `source_lang`
* Новый `target_lang` (соответствующий кнопке)

То есть если изначально выбрано **Русский → Испанский**, можно одной кнопкой перейти, например, в **Русский → Английский** для того же слова.
Открывается новая страница результатов с новой парой языков.

Для поля поиска может использоваться endpoint автодополнения:
